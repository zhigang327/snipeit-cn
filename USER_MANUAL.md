## 系统概述

Snipe-CN 是一个功能完善的IT资产管理系统,帮助企业有效管理和追踪IT资产的全生命周期。

### 主要功能
- **多级部门管理**: 支持无限级部门层级结构
- **资产管理**: 资产入库、分配、归还、盘点、报废
- **用户管理**: 用户账号、角色、权限管理
- **数据统计**: 丰富的图表和报表
- **历史追溯**: 完整的操作历史记录

## 登录与权限

### 系统登录

1. 打开浏览器,访问系统地址
2. 输入邮箱和密码
3. 点击"登录"按钮

**默认管理员账号**:
- 邮箱: `admin@example.com`
- 密码: `admin123`

⚠️ **安全提示**: 首次登录后请立即修改默认密码!

### 权限说明

系统采用基于角色的权限控制(RBAC):

#### 管理员权限 (Admin)
- 部门管理: 创建、编辑、删除部门
- 资产管理: 所有操作权限
- 用户管理: 创建、编辑、删除用户
- 系统设置: 访问所有系统配置

#### 普通用户权限 (User)
- 查看资产列表
- 查看自己名下的资产
- 申请资产领用
- 归还已领用的资产

## 部门管理

### 查看部门树

点击左侧菜单"部门管理",可以看到整个组织的部门树形结构:

```
总公司
├── 技术部
│   ├── 后端开发组
│   └── 前端开发组
├── 人力资源部
└── 财务部
```

### 添加部门

1. 点击右上角"添加部门"按钮
2. 填写部门信息:
   - **部门名称**: 必填,例如"技术部"
   - **部门编码**: 必填,唯一标识,例如"TECH"
   - **父部门**: 可选,选择上级部门
   - **负责人**: 可选,选择部门负责人
   - **排序**: 数字,越小越靠前
   - **地址**: 可选,部门地址
   - **电话**: 可选,部门联系电话
   - **状态**: 启用/禁用
   - **描述**: 可选,部门描述

3. 点击"确定"保存

### 编辑部门

1. 在部门列表中找到目标部门
2. 点击"编辑"按钮
3. 修改需要更新的信息
4. 点击"确定"保存

### 删除部门

⚠️ **注意**: 只有以下条件的部门才能被删除:
- 没有子部门
- 没有归属用户

删除步骤:
1. 点击部门行的"删除"按钮
2. 确认删除操作

### 移动部门

部门可以移动到不同的父部门下,调整组织结构:

1. 编辑部门
2. 修改"父部门"字段
3. 系统会自动检测并防止循环引用

### 部门统计

点击部门可以查看该部门的详细统计信息:
- 用户数量
- 资产数量
- 总资产价值
- 子部门数量
- 部门层级深度

## 资产管理

### 查看资产列表

点击左侧菜单"资产管理",显示所有资产列表,支持:

**搜索功能**:
- 支持按资产名称、标签、序列号搜索
- 按状态筛选(在库、已分配、维修中、已损坏等)

**列表信息**:
- 资产标签: 唯一标识,如 AST-0001
- 资产名称: 资产的正式名称
- 分类: 资产所属分类
- 品牌/型号: 资产的品牌和型号
- 使用人: 当前使用该资产的用户
- 部门: 资产所属部门
- 状态: 资产当前状态
- 采购价格: 资产的采购价格

### 添加资产

1. 点击右上角"添加资产"按钮
2. 填写资产信息:

**基本信息**:
- **资产标签**: 必填,唯一标识,格式如 AST-0001
- **资产名称**: 必填,例如 "Dell Latitude 5420 笔记本电脑"
- **分类**: 必填,选择资产分类
- **供应商**: 可选,选择供应商

**设备信息**:
- **品牌**: 例如 Dell, HP, Lenovo
- **型号**: 具体型号
- **序列号**: 设备SN码
- **采购日期**: 采购时间

**财务信息**:
- **采购价格**: 资产购买价格(元)
- **保修期**: 保修月数,默认12个月
  - 系统会自动计算保修到期日期

**存放信息**:
- **存放位置**: 例如 "A楼3层305室"
- **部门**: 可选,指定资产所属部门

**其他信息**:
- **备注**: 任意备注信息

3. 点击"确定"保存

### 编辑资产

1. 在资产列表中找到目标资产
2. 点击"编辑"按钮
3. 修改需要更新的信息
4. 点击"确定"保存

系统会自动记录修改历史,可追溯所有变更。

### 分配资产

将资产分配给用户使用:

1. 在资产列表中,找到状态为"在库"的资产
2. 点击"分配"按钮
3. 填写分配信息:
   - **使用人**: 必填,选择使用该资产的用户
   - **部门**: 可选,默认使用用户所在部门
   - **预计归还**: 可选,设置预计归还日期
   - **备注**: 分配备注
4. 点击"确定"

分配后:
- 资产状态变为"已分配"
- 记录分配操作历史
- 用户可在个人中心查看名下资产

### 归还资产

用户归还资产:

1. 在资产列表中,找到状态为"已分配"的资产
2. 点击"归还"按钮
3. 填写归还信息:
   - **存放位置**: 归还后存放的位置
   - **备注**: 归还备注
4. 点击"确定"

归还后:
- 资产状态变为"在库"
- 清除使用人信息
- 记录归还操作历史

### 删除资产

⚠️ **注意**: 只有状态为"在库"的资产才能被删除

删除步骤:
1. 点击资产行的"删除"按钮
2. 确认删除操作

删除会:
- 记录删除历史
- 软删除数据(可恢复)

### 资产状态说明

| 状态 | 说明 | 可操作 |
|------|------|--------|
| 在库 | 资产在库,可分配 | 分配、编辑、删除 |
| 已分配 | 资产已分配给用户 | 归还、查看 |
| 维修中 | 资产在维修 | 编辑、查看 |
| 已损坏 | 资产已损坏 | 编辑、报废 |
| 已丢失 | 资产已丢失 | 编辑、报废 |
| 已报废 | 资产已报废 | 仅查看 |

### 资产历史记录

查看资产的所有操作历史:

1. 点击资产行的"查看"按钮
2. 在详情中查看"操作历史"

历史记录包括:
- 操作时间
- 操作人
- 操作类型(创建、分配、归还、更新等)
- 变更前后的值
- 操作备注

## 用户管理

### 查看用户列表

点击左侧菜单"用户管理",显示所有用户列表。

### 添加用户

1. 点击"添加用户"按钮
2. 填写用户信息:
   - **姓名**: 用户真实姓名
   - **邮箱**: 用于登录的邮箱,必须唯一
   - **密码**: 登录密码
   - **电话**: 联系电话
   - **部门**: 所属部门
   - **工号**: 员工工号
   - **职位**: 职位名称
   - **入职日期**: 入职时间

3. 选择用户角色(管理员/普通用户)
4. 点击"确定"保存

### 编辑用户

1. 在用户列表中找到目标用户
2. 点击"编辑"按钮
3. 修改需要更新的信息
4. 点击"确定"保存

### 禁用/启用用户

编辑用户时,可以设置"是否启用"状态:
- 启用: 用户可以正常登录
- 禁用: 用户无法登录系统

## 数据统计

### 仪表盘概览

登录系统后默认进入仪表盘,可以看到:

**资产统计卡片**:
- 总资产数: 系统中所有资产总数

**资产价值统计**:
- 总资产价值: 所有资产的采购总价
- 平均资产价值: 单个资产的平均价格

### 部门统计

在部门管理中,点击任意部门可以查看:
- 部门下的用户数量(包括所有子部门)
- 部门下的资产数量(包括所有子部门)
- 部门资产总价值(包括所有子部门)

## 最佳实践

### 部门设置建议

1. **层级设计**: 建议不超过4-5级,避免过深层级
2. **编码规范**: 使用简短、有意义的编码,如:
   - 根部门: ROOT
   - 技术部: TECH
   - 技术部-后端: TECH-BE
3. **负责人**: 每个部门都应设置负责人
4. **命名一致**: 使用统一的命名规范

### 资产管理建议

1. **标签规则**: 建立统一的资产标签规则
   - 格式: 类型-年份-序号,如 PC-2024-0001
   - 或: 缩写+流水号,如 AST0001

2. **分类体系**: 建立合理的资产分类体系
   - 电脑设备(笔记本、台式机、服务器)
   - 网络设备(路由器、交换机)
   - 外设设备(显示器、键盘、鼠标)
   - 办公设备(打印机、扫描仪)

3. **定期盘点**: 建议每季度进行一次资产盘点

4. **及时更新**: 资产状态变更时及时在系统中更新

5. **图片管理**: 为重要资产拍摄照片上传

### 安全建议

1. **密码策略**:
   - 定期修改密码(建议每季度)
   - 使用复杂密码(大小写字母+数字+符号)
   - 不要与其他系统共用密码

2. **权限最小化**:
   - 按需分配权限
   - 定期审查用户权限
   - 离职员工及时禁用账号

3. **数据备份**:
   - 定期备份数据库
   - 保留多个历史备份
   - 测试备份恢复流程

## 常见问题

### Q1: 忘记密码怎么办?
**A**: 联系系统管理员重置密码。管理员可以通过命令行重置:
```bash
docker-compose exec backend php artisan tinker
>>> $user = \App\Models\User::where('email', 'your@email.com')->first()
>>> $user->password = bcrypt('new_password')
>>> $user->save()
```

### Q2: 如何导出资产数据?
**A**: 目前可通过查询筛选后,使用浏览器的打印功能保存为PDF,或联系开发人员添加Excel导出功能。

### Q3: 资产分配后可以修改吗?
**A**: 可以。先归还资产,然后重新分配。或者联系管理员直接在后台修改。

### Q4: 如何查看自己名下的资产?
**A**: 登录后,在资产管理页面筛选"使用人"为当前用户。

### Q5: 删除的资产可以恢复吗?
**A**: 系统使用软删除,删除的数据可以恢复。联系数据库管理员执行恢复操作。

### Q6: 部门可以设置多个负责人吗?
**A**: 当前版本只支持单个负责人。如需多人管理,建议:
- 设置主负责人
- 其他负责人通过角色权限管理

### Q7: 资产分类可以修改吗?
**A**: 可以编辑分类信息。建议不要频繁修改分类体系,以免影响历史数据统计。

### Q8: 如何批量添加资产?
**A**: 当前版本需要逐个添加。后续版本会支持Excel批量导入功能。

### Q9: 系统支持条码/二维码吗?
**A**: 系统为每个资产生成二维码标签。可以打印后贴在资产上,通过扫码快速查看资产信息。

### Q10: 如何设置资产到期提醒?
**A**: 在编辑资产时,设置"预计归还日期"。系统会在到期前发送邮件提醒(需配置邮件服务)。
