# 二维码和扫码盘点功能说明

## 功能概述

Snipe-CN 支持完整的二维码管理和PDA扫码盘点功能,帮助企业高效管理资产。

## 二维码功能

### 1. 生成资产二维码

每个资产都可以生成唯一的二维码,二维码包含资产访问链接。

**API接口**:
```http
POST /api/assets/{id}/qrcode
```

**生成内容**:
- 二维码包含资产的访问URL
- 格式: `http://yourdomain.com/api/assets/{id}`
- 扫描后可直接查看资产详情

**使用方法**:

#### 方式一: 单个生成
```bash
# 通过API
curl -X POST http://localhost:8000/api/assets/1/qrcode \
  -H "Authorization: Bearer YOUR_TOKEN"
```

#### 方式二: 批量生成
```bash
# 批量生成多个资产的二维码
curl -X POST http://localhost:8000/api/assets/qrcode/batch \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "asset_ids": [1, 2, 3, 4, 5]
  }'
```

#### 方式三: 前端生成(推荐)
1. 进入资产管理页面
2. 勾选需要生成二维码的资产
3. 点击"批量生成二维码"按钮
4. 系统自动生成并更新数据库

### 2. 下载二维码

**API接口**:
```http
GET /api/assets/{id}/qrcode/download
```

**使用方法**:
```bash
# 下载单个资产的二维码
curl -O http://localhost:8000/api/assets/1/qrcode/download \
  -H "Authorization: Bearer YOUR_TOKEN"
```

下载的文件名格式: `{资产标签}_qrcode.png`

### 3. 打印资产标签

系统支持生成打印友好的资产标签HTML。

**API接口**:
```http
GET /api/assets/{id}/qrcode/print
```

**标签内容**:
- 资产二维码
- 资产标签
- 资产名称
- 资产分类
- 资产型号

**打印建议**:
1. 标签尺寸: 50mm x 30mm 或 100mm x 60mm
2. 纸张: 不干胶标签纸
3. 打印机: 条码打印机(如Zebra、TSC等)
4. 推荐分辨率: 300dpi

## 扫码盘点功能

### 1. 创建盘点任务

**API接口**:
```http
POST /api/inventories
```

**请求参数**:
```json
{
  "name": "2024年第一季度盘点",
  "description": "技术部门资产盘点",
  "department_id": 1
}
```

**使用方法**:
1. 进入"资产盘点"页面
2. 点击"开始盘点"按钮
3. 填写盘点信息:
   - 盘点名称
   - 盘点部门(可选)
   - 备注
4. 点击"确定"

### 2. PDA扫码盘点

**支持的设备**:
- PDA手持终端(推荐)
- 智能手机
- 平板电脑
- 带摄像头的笔记本电脑

**盘点流程**:

#### 方式一: 手动输入资产标签
1. 打开盘点任务
2. 点击"扫码盘点"按钮
3. 手动输入资产标签
4. 确认盘点状态:
   - 正常
   - 未找到
   - 丢失
   - 损坏
5. 填写实际位置和备注
6. 点击"确认"

#### 方式二: 扫描二维码
1. 打开盘点任务
2. 点击"打开摄像头"按钮
3. 使用PDA或手机的摄像头扫描资产上的二维码
4. 系统自动识别并显示资产信息
5. 确认盘点状态
6. 点击"确认"

**API接口**:
```http
POST /api/inventories/{id}/scan
```

**请求参数**:
```json
{
  "asset_tag": "AST-0001",
  "status": "found",
  "location": "A楼3层305室",
  "notes": "设备完好"
}
```

**盘点状态说明**:
- `found`: 资产正常找到
- `not_found`: 预期位置未找到
- `lost`: 确认丢失
- `damaged`: 资产损坏

### 3. 查看盘点进度

**API接口**:
```http
GET /api/inventories/{id}/progress
```

**返回数据**:
```json
{
  "success": true,
  "data": {
    "total": 100,
    "found": 95,
    "lost": 3,
    "damaged": 2,
    "not_found": 0,
    "progress": 95.0
  }
}
```

### 4. 完成盘点

**API接口**:
```http
POST /api/inventories/{id}/complete
```

**请求参数**:
```json
{
  "notes": "盘点完成,发现2台设备损坏"
}
```

完成盘点后,系统会:
1. 更新盘点状态为"已完成"
2. 自动更新资产状态:
   - 丢失的资产状态改为"lost"
   - 损坏的资产状态改为"broken"
3. 记录完成时间

## PDA设备推荐

### 入门级PDA
- **品牌**: Newland, Unitech
- **特点**: 价格实惠,基本功能
- **适用**: 小型企业盘点

### 专业级PDA
- **品牌**: Zebra, Honeywell
- **特点**: 扫码速度快,耐用性强
- **适用**: 大型企业高频盘点

### 智能手机方案
- **设备**: Android/iOS手机 + 扫码APP
- **特点**: 利用现有设备,成本低
- **适用**: 中小型企业

## 使用场景

### 场景1: 日常资产查询
1. 用手机扫描资产上的二维码
2. 立即查看资产详细信息
3. 查看资产当前状态和使用人

### 场景2: 资产借还登记
1. 借出时扫描二维码
2. 系统自动分配给当前用户
3. 归还时再次扫描
4. 系统自动更新状态为"在库"

### 场景3: 定期资产盘点
1. 创建盘点任务
2. 用PDA逐个扫描资产二维码
3. 标记异常资产(丢失/损坏)
4. 完成盘点后生成报告

### 场景4: 资产位置变更
1. 扫描资产二维码
2. 更新资产位置信息
3. 系统记录位置变更历史

## 扫码兼容性

### 支持的二维码内容
1. **资产ID链接**: `http://domain.com/api/assets/1`
2. **资产标签**: `AST-0001`
3. **资产编码**: 自定义编码

### 扫码方式
- **摄像头扫描**: 实时摄像头识别
- **条码枪扫描**: 硬件条码枪输入
- **手动输入**: 直接输入资产标签

## 技术实现

### 二维码生成
- **库**: Simple QrCode (PHP)
- **格式**: PNG
- **大小**: 300x300px
- **纠错级别**: High (30%恢复)

### 扫码识别
- **前端摄像头**: 原生Camera API
- **二维码解析**: 可集成jsQR等库
- **条码枪**: 标准键盘输入

### 数据存储
- **存储位置**: `storage/app/public/qr_codes/`
- **文件名**: `asset_{id}_{timestamp}.png`
- **URL**: `/storage/qr_codes/{filename}`

## 常见问题

### Q1: 二维码无法识别?
**A**:
- 检查二维码是否清晰
- 确认扫码设备支持二维码
- 尝试调整扫描角度和距离
- 必要时重新生成二维码

### Q2: PDA无法连接系统?
**A**:
- 检查网络连接
- 确认URL配置正确
- 检查防火墙设置
- 测试API是否可访问

### Q3: 盘点进度不准确?
**A**:
- 确保每次扫码都成功提交
- 刷新进度页面查看最新数据
- 检查是否有重复扫描
- 联系管理员核对数据

### Q4: 打印的二维码质量差?
**A**:
- 使用专业条码打印机
- 选择合适的打印分辨率(300dpi+)
- 使用质量好的不干胶纸
- 调整二维码大小(建议50mm以上)

### Q5: 如何集成其他扫码设备?
**A**:
- 大多数扫码枪模拟键盘输入
- 无需特殊驱动,即插即用
- 扫码结果会自动输入到文本框
- 支持配置扫码后自动提交
