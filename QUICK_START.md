# Snipe-CN 快速入门指南

## 5分钟快速上手

### 步骤1: 登录系统

访问系统地址,使用默认管理员账号登录:
- 邮箱: `admin@example.com`
- 密码: `admin123`

⚠️ 首次登录后请立即修改密码!

### 步骤2: 创建组织架构

1. 点击左侧菜单 **"部门管理"**
2. 点击 **"添加部门"** 按钮创建根部门:
   - 部门名称: `总公司`
   - 部门编码: `ROOT`
   - 其他保持默认
3. 继续创建子部门:
   - 选择"总公司"作为父部门
   - 创建"技术部",编码 `TECH`
   - 创建"人力资源部",编码 `HR`

### 步骤3: 添加用户

1. 点击左侧菜单 **"用户管理"**
2. 点击 **"添加用户"** 创建用户:
   - 姓名: `张三`
   - 邮箱: `zhangsan@company.com`
   - 密码: `123456`
   - 部门: 选择 `技术部`
   - 角色: 选择 `普通用户`

### 步骤4: 录入资产

1. 点击左侧菜单 **"资产管理"**
2. 点击 **"添加资产"**:
   - 资产标签: `AST-0001`
   - 资产名称: `Dell Latitude 5420`
   - 分类: `电脑设备 > 笔记本`
   - 品牌: `Dell`
   - 型号: `Latitude 5420`
   - 采购价格: `8000`
   - 采购日期: 选择今天的日期
   - 存放位置: `A楼3层`
3. 点击 **"确定"** 保存

### 步骤5: 分配资产

1. 在资产列表中,找到刚创建的资产
2. 点击 **"分配"** 按钮
3. 选择使用人: `张三`
4. 选择部门: `技术部` (自动填充)
5. 点击 **"确定"**

完成! 现在:
- 资产状态变为"已分配"
- 张三名下有了这个资产
- 可以在仪表盘查看统计信息

## 常用场景操作

### 场景1: 员工离职归还资产

1. 进入资产管理
2. 筛选使用人,找到离职员工的资产
3. 逐个点击"归还"按钮
4. 更新存放位置
5. 在用户管理中禁用该用户账号

### 场景2: 新员工领用设备

1. 确认设备在库
2. 点击"分配"按钮
3. 选择新员工作为使用人
4. 设置预计归还日期
5. 保存

### 场景3: 资产盘点

1. 进入资产管理
2. 逐一核对实际资产
3. 更新状态(损坏/丢失的在库资产)
4. 标记找到的丢失资产
5. 记录盘点结果到备注

### 场景4: 创建新部门

1. 进入部门管理
2. 添加部门,填写基本信息
3. 设置部门负责人
4. 为部门添加员工
5. 将资产分配给部门

### 场景5: 资产转移

1. 找到要转移的资产
2. 如果已分配,先"归还"
3. 重新"分配"给新的用户/部门

## 数据流程图

### 资产生命周期

```
采购入库 → 在库 → 分配给用户 → 已分配 → 归还 → 在库
                       ↓
                     维修中 → 维修完成 → 在库
                       ↓
                     已损坏 → 报废 → 已报废
                       ↓
                     已丢失 → 报废 → 已报废
```

### 权限流程

```
登录系统 → 验证凭证 → 获取用户角色 → 检查权限 → 访问功能
                                              ↓
                    权限不足 → 返回无权访问
```

## 关键概念

### 多级部门

Snipe-CN 支持无限级部门嵌套,例如:

```
总公司 (ROOT)
├── 技术部 (TECH)
│   ├── 后端开发组 (TECH-BE)
│   │   └── 后端一组 (TECH-BE-1)
│   └── 前端开发组 (TECH-FE)
├── 人力资源部 (HR)
└── 财务部 (FINANCE)
```

**特点**:
- 支持任意层级深度
- 每个部门可以设置负责人
- 可以统计部门及其所有子部门的数据
- 部门可以移动到其他父部门下

### 资产标签

每个资产都有唯一的标签,用于标识和追踪:

- 格式: 自定义,如 `AST-0001`、`PC-2024-001`
- 作用: 扫码识别、快速搜索
- 建议: 建立统一的编码规则

### 资产状态

资产状态反映了资产当前的流转状态:

| 状态 | 图标 | 说明 |
|------|------|------|
| 在库 | ✅ | 资产在库,可分配 |
| 已分配 | 🔵 | 已分配给用户使用 |
| 维修中 | ⚠️ | 正在维修中 |
| 已损坏 | ❌ | 已损坏,待处理 |
| 已丢失 | ⚫ | 已丢失 |
| 已报废 | 🗑️ | 已报废,不可使用 |

### 操作历史

系统记录所有对资产的操作,包括:
- 创建资产
- 分配/归还
- 更新信息
- 删除(软删除)

可以追溯完整的资产流转过程。

## 高级功能

### 1. 部门统计

点击任意部门,可以查看:
- 用户数量(含子部门)
- 资产数量(含子部门)
- 总资产价值(含子部门)
- 子部门数量
- 层级深度

### 2. 资产搜索

支持多维度搜索:
- 资产名称
- 资产标签
- 序列号
- 型号
- 状态筛选

### 3. 二维码

每个资产自动生成二维码,可用于:
- 打印标签贴在资产上
- 扫码快速查看资产信息
- 资产盘点时扫码核对

### 4. 保修管理

录入采购日期和保修期后:
- 自动计算保修到期日
- 保修到期前发送提醒(需配置邮件)
- 方便追踪质保情况

### 5. 历史追溯

完整记录资产操作历史:
- 操作时间
- 操作人
- 操作类型
- 变更前后的值
- 备注信息
