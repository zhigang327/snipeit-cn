## ✨ 新功能

### 资产维修记录管理

全新的资产维修记录管理功能，帮助您更好地追踪和管理资产的维修历史。

#### 核心功能

1. **维修记录CRUD**
   - 创建维修记录
   - 编辑维修信息
   - 查看维修详情
   - 删除维修记录（管理员）

2. **维修状态管理**
   - 待处理 (Pending)
   - 处理中 (In Progress)
   - 已完成 (Completed)
   - 已取消 (Cancelled)

3. **维修优先级**
   - 低 (Low)
   - 中 (Medium)
   - 高 (High)
   - 紧急 (Urgent)

4. **维修类型**
   - 硬件 (Hardware)
   - 软件 (Software)
   - 网络 (Network)
   - 其他 (Other)

5. **维修人员分配**
   - 分配维修负责人
   - 自动状态更新
   - 权限控制

6. **成本统计**
   - 预估工时和费用
   - 实际工时和费用
   - 维修成本分析

7. **统计面板**
   - 总维修记录数
   - 待处理数量
   - 处理中数量
   - 完成率统计

8. **逾期提醒**
   - 自动识别逾期维修
   - 超过7天标记为逾期
   - 便于及时处理

## 📋 使用指南

### 快速开始

1. **创建维修记录**
   ```
   菜单: 维修记录 → 新增维修记录
   步骤:
   1. 选择需要维修的资产
   2. 填写维修标题和故障描述
   3. 设置优先级和维修类型
   4. 预估工时和费用
   5. 保存
   ```

2. **分配维修人员**
   ```
   在维修记录列表中:
   1. 找到状态为"待处理"的记录
   2. 点击"分配"按钮
   3. 选择维修人员
   4. 确认分配
   ```

3. **完成维修**
   ```
   在维修记录列表中:
   1. 找到状态为"处理中"的记录
   2. 点击"完成"按钮
   3. 填写故障诊断和解决方案
   4. 记录实际工时和费用
   5. 确认完成
   ```

### API接口

```
GET    /api/maintenance              # 获取维修记录列表
POST   /api/maintenance              # 创建维修记录
GET    /api/maintenance/{id}         # 获取维修记录详情
PUT    /api/maintenance/{id}         # 更新维修记录
DELETE /api/maintenance/{id}         # 删除维修记录
POST   /api/maintenance/{id}/assign  # 分配维修人员
POST   /api/maintenance/{id}/complete # 完成维修
POST   /api/maintenance/{id}/cancel  # 取消维修
GET    /api/maintenance/statistics   # 获取统计信息
GET    /api/maintenance/overdue      # 获取逾期记录
GET    /api/assets/{asset}/maintenance/history # 获取资产维修历史
```
